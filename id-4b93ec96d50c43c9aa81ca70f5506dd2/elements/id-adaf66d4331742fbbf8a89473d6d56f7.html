Docker light base software components Docker managed ()
Container components implementing direct process execution where the main application process runs as PID 1, delegating lifecycle management to Docker Engine. These containers follow the single-process-per-container principle with external supervision handled by Docker's restart policies. Service failures result in container termination and restart, managed entirely by Docker Engine orchestration. Startup sequence includes direct entrypoint execution, environment variable processing, and immediate application launch without internal supervision layers.
