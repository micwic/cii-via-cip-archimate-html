The **`pom.xml`** file is the core configuration file in a Maven project. It stands for **Project Object Model** and defines the structure, dependencies, and build instructions for the project. ### Key Features of `pom.xml`: 1. **Project Information**: - Provides metadata about the project, such as its name, description, version, and group ID. - Example: ```xml <groupId>com.example</groupId> <artifactId>my-app</artifactId> <version>1.0.0</version> ``` 2. **Dependencies**: - Lists the libraries and other projects the application depends on. - Maven automatically downloads these dependencies from a repository. - Example: ```xml <dependencies> <dependency> <groupId>org.springframework</groupId> <artifactId>spring-core</artifactId> <version>5.3.8</version> </dependency> </dependencies> ``` 3. **Plugins**: - Defines plugins used during the build process, such as compilers, test frameworks, or deployment tools. - Example: ```xml <build> <plugins> <plugin> <groupId>org.apache.maven.plugins</groupId> <artifactId>maven-compiler-plugin</artifactId> <version>3.8.1</version> <configuration> <source>1.8</source> <target>1.8</target> </configuration> </plugin> </plugins> </build> ``` 4. **Repositories**: - Specifies the locations from which Maven fetches dependencies (default is Maven Central Repository). - Example: ```xml <repositories> <repository> <id>central</id> <url>https://repo.maven.apache.org/maven2</url> </repository> </repositories> ``` 5. **Build Configuration**: - Customizes the build process, including specific instructions for packaging, testing, or deployment. - Example: ```xml <packaging>jar</packaging> ``` 6. **Profiles**: - Allows defining different configurations for various environments (e.g., development, testing, production). - Example: ```xml <profiles> <profile> <id>production</id> <activation> <activeByDefault>true</activeByDefault> </activation> <properties> <env>production</env> </properties> </profile> </profiles> ``` ### Purpose: - Automates the process of building, testing, and deploying applications. - Ensures dependency management is consistent and repeatable across environments. - Simplifies project configuration for multi-module projects. ### Common Commands: - `mvn clean install`: Cleans the project and installs the packaged application in the local repository. - `mvn dependency:tree`: Displays the dependency hierarchy for troubleshooting. The `pom.xml` file is central to any Maven project and enables developers to manage their project's lifecycle efficiently.
