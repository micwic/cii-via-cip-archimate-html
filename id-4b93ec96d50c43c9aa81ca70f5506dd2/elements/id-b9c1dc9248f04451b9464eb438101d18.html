Create/use existing .m2 directory in your personnal settings or in the projet directory (need to be configured into Maven). 5. Set user/password into settings.xml (settings-security.xml) to flowable artefacts repository by using the security strategy defined for your context (password, maven encrypted password, identity token from JFrog.com) 6. Generate Spring Boot Skeleton for flowable by using start.flowable.com or start.spring.io, download Spring Boot Skeleton and Unzip it into the CII via Cip source code folder 7. Adjust pom.xml to what we need as source code, we need only flowable work. Note that we will remove Flowable Work from the standard Flowable distribution for Docker as it will be executed through IntelliJ in run or debug mode to meet the requirements of ongoing customization. 8. Adjust java Main class and add security customized java classes 9. Create User Definitions in the json configuration file 10. Adjust application.properties configuration file (Inspect, Database, ElasticSearch, ...) 11. Get flowable source code from JFROG flowable repository by using Maven command 12. Launch Docker flowable other dependant components 13. Build and Run or Debug the main application class in IntelliJ 14. Add changes to git, Commit and Push everything into github project repository To do: Extend to functional and infrastructure testing as proposed in the Flowable documentation (https://documentation.flowable.com/latest/develop/be/java-extensions#test-a-custom-project)
